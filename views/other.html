Other API's

These are a variety of other API's available that we'll skip for now.

  • screen

    Provides information about screen size, displays, cursor position, etc.

  • shell

    Provides functions related to desktop integration.

  • crash-reporter

    Enables sending your app's crash reports.

  • session

    Enables access cookies, proxy settings, network emulation, etc.

  • protocol

    Enables registering a custom protocol or intercept an existing protocol.

  • webContents

    Enables you to access parts of the web contents of a browser window from the main process.

  • content-tracing

    This is used to collect tracing data generated by the underlying Chromium content module.

  • File object

    The DOM's File interface provides abstraction around native files in order to let users work on native files directly with the HTML5 file API.

  • <webview> tag

    Unlike an iframe, the webview runs in a separate process than your app. It doesn't have the same permissions as your web page and all interactions between your app and embedded content will be asynchronous. This keeps your app safe from the embedded content.
